# Sequelize Global Default Hooks ## Background In Sequelize, you can define global hooks that apply to all models unless overridden. This is useful for automatically setting custom timestamp formats or audit fields across your entire application without defining them on every single model. ## Requirements * Create a Node.js script that initializes a Sequelize instance with SQLite. * Configure a global default `beforeCreate` hook on the Sequelize constructor (using the `define` option) that automatically sets a custom field `auditLog` to the string `"Created via global hook"` on any model being created. * Define two models: `User` and `Project`. * The `Project` model should override the `beforeCreate` hook and set `auditLog` to `"Created via local hook"`. * Create one `User` and one `Project` record and save them to the database. * Write the resulting records to a JSON file. ## Implementation Hints * Use the `define` option in the `Sequelize` constructor to add default hooks. * Ensure both models have an `auditLog` string column defined. * Remember to sync the database before creating records. * Write the output of `User.findAll()` and `Project.findAll()` to `output.json`. ## Acceptance Criteria * Project path: /home/user/myproject * Ensure the script is executed and the artifacts exist. * Command: `node index.js` * The script must create a SQLite database file at `/home/user/myproject/database.sqlite`. * Log file: /home/user/myproject/output.json * The `output.json` file must contain an object with a `users` array and a `projects` array. * The `User` record in `output.json` must have `auditLog` set to `"Created via global hook"`. * The `Project` record in `output.json` must have `auditLog` set to `"Created via local hook"`.
